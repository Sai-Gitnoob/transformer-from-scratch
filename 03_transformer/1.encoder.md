# Transformer Encoder Architecture

## 1. Overview

The Transformer encoder is composed of **N = 6 identical layers stacked sequentially**.  
Each layer (also called an *encoder block*) has the same internal structure but learns different representations during training.

Each encoder block contains two main sublayers:

1. **Multi-Head Self-Attention**
2. **Feed Forward Neural Network**

---

## 2. High-Level Flow

```
Input
↓
[Encoder Block 1]
↓
[Encoder Block 2]
↓
...
↓
[Encoder Block 6]
↓
Final Encoder Output (z)

```
---

Each block refines the representation produced by the previous one.

---

## 3. Structure of a Single Encoder Block

Each encoder block consists of two sublayers, and **each sublayer is wrapped with:**

- A **residual connection**
- Followed by **layer normalization**

### Substructure:
```
Input (x)
↓
Multi-Head Self-Attention
↓
LayerNorm(x + Sublayer(x))
↓
Feed Forward Network
↓
LayerNorm(x + Sublayer(x))

```
---

---

## 4. Important Clarification

- **N = 6** means there are **6 identical encoder blocks stacked**
- It does NOT mean:
  - Layer 1 = Self-Attention
  - Layer 2 = Feed Forward

Instead:

> Each of the 6 blocks contains both sublayers.

---

## 5. Why Stack Multiple Layers?

Stacking layers allows the model to progressively refine its understanding.

You can think of it as multiple passes over the same sentence:

- Pass 1: Basic structure
- Pass 2: Relationships between words
- Pass 3: Context understanding
- Pass 4: Deeper dependencies
- Pass 5: Subtle patterns
- Pass 6: Final refined representation

Each layer builds on the previous one.

---

## 6. Residual Connections

Each sublayer uses a residual connection:

\[
\text{Output} = x + \text{Sublayer}(x)
\]

### Purpose:

- Prevents loss of original information
- Helps gradients flow during training
- Enables deeper networks without degradation

### Intuition:

Instead of replacing the input, the model **adds new information on top of it**.

---

## 7. Layer Normalization

After each residual addition, layer normalization is applied:

\[
\text{LayerNorm}(x + \text{Sublayer}(x))
\]

### Purpose:

- Stabilizes training
- Keeps values within a consistent range
- Ensures each layer receives well-conditioned inputs

The sequence is always:

> Add → Normalize

---

## 8. Model Dimension (d_model = 512)

All vectors in the encoder have the same dimensionality:

\[
d_{\text{model}} = 512
\]

This includes:

- Input embeddings
- Attention outputs
- Feed-forward outputs

### Why is this important?

Residual connections require:

\[
x + \text{Sublayer}(x)
\]

For this addition to work:

> Both terms must have the same dimensionality.

This enforces a consistent representation space throughout the model.

---

## 9. Final Summary

- The encoder consists of **6 identical stacked blocks**
- Each block contains:
  - Multi-head self-attention
  - Feed-forward network
- Each sublayer uses:
  - Residual connection
  - Layer normalization
- All representations maintain a fixed dimension of **512**

---

## 10. One-Line Summary

Six identical encoder blocks progressively refine the input representation using attention and feed-forward layers, while residual connections preserve information and layer normalization ensures stability.
