# Transformer Decoder Architecture

## 1. Overview

The Transformer decoder is structurally similar to the encoder:

- It consists of **N = 6 identical stacked layers**
- Each layer refines the representation progressively

However, the decoder introduces two key differences:

1. An additional **cross-attention sublayer**
2. A modification to self-attention called **masked self-attention**

---

## 2. High-Level Flow

```
Encoder Output (z)
↓
↓ ─────────────────────────────┐
│
[Decoder Block 1] │
├── Sublayer 1: Masked Multi-Head │
│ Self-Attention │
│ ↓ │
│ LayerNorm(x + Sublayer(x)) │
│ │
├── Sublayer 2: Cross-Attention ←───┘
│ (Encoder-Decoder Attention)
│ ↓
│ LayerNorm(x + Sublayer(x))
│
├── Sublayer 3: Feed Forward Network
│ ↓
│ LayerNorm(x + Sublayer(x))
↓
[Decoder Block 2]
... same structure
↓
[Decoder Block 6]
↓
Final Output
```

## 3. Structure of a Decoder Block

Each decoder block contains **three sublayers**:

1. **Masked Multi-Head Self-Attention**
2. **Cross-Attention (Encoder–Decoder Attention)**
3. **Feed Forward Network**

Each sublayer is wrapped with:

- A **residual connection**
- Followed by **layer normalization**

---

## 4. Cross-Attention (Encoder–Decoder Attention)

This is the additional sublayer compared to the encoder.

### Purpose:

- Allows the decoder to use the encoder’s output representation \( z \)
- Helps the decoder focus on relevant parts of the input while generating output

### Intuition:

- The encoder produces a complete understanding of the input
- The decoder refers back to this understanding at every step

### Conceptual View:

> The decoder asks:  
> "Given what I have generated so far, which parts of the input are important now?"

---

## 5. Masked Self-Attention

This is a modified version of self-attention used only in the decoder.

### Problem:

During training, the model could "see" future words and learn to copy instead of learning meaningful patterns.

### Solution:

- Apply a **mask** to hide future positions
- Each position can only attend to:
  - Itself
  - Previous positions

### Effect:

| Step | Can See                | Cannot See      |
|------|------------------------|-----------------|
| 1    | [START]               | Future words    |
| 2    | [START], word₁        | Future words    |
| 3    | [START], word₁, word₂ | Future words    |

This ensures:

> Position *i* depends only on positions **less than or equal to i**

---

## 6. Output Offset (Auto-Regressive Setup)

During training, the decoder input is **shifted by one position**.

### Example:

| Decoder Input            | Expected Output |
|--------------------------|-----------------|
| [START]                  | word₁           |
| [START], word₁           | word₂           |
| [START], word₁, word₂    | word₃           |

### Why this matters:

- Enforces **auto-regressive generation**
- Ensures predictions depend only on **previous outputs**

---

## 7. Encoder vs Decoder Comparison


| Component              | Encoder                     | Decoder                              |
|------------------------|----------------------------|--------------------------------------|
| Number of Layers       | 6                          | 6                                    |
| Sublayer 1             | Self-Attention             | Masked Self-Attention                |
| Sublayer 2             | Feed Forward               | Cross-Attention (with encoder)       |
| Sublayer 3             | —                          | Feed Forward                         |
| Residual + Norm        | Yes                        | Yes                                  |
| Can See Future Tokens  | Yes                        | No (masked)                          |
| Model Dimension        | 512                        | 512                                  |

## 8. Key Takeaways
Decoder = Encoder + additional cross-attention layer
Masking ensures no access to future tokens
Output is generated step-by-step (auto-regressive)
Residual connections and normalization ensure stable training
All representations remain in the same dimensional space

---

## 9. One-Line Summary

The decoder extends the encoder by adding cross-attention to access input information and masked self-attention to ensure step-by-step generation without using future tokens.
