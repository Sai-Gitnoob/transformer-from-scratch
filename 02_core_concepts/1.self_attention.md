## Scaled Dot-Product Attention

### The Formula

$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

This is computed in four steps. Consider the sentence:

> "The cat sat on the mat because it was tired"

The task is to figure out what "it" refers to.

---

### Step 1: Compute the Dot Product of Q and K

The query for "it" is compared against every key in the sequence:

```
"it" · "cat"   = 8.5   (high match)
"it" · "mat"   = 3.2   (low match)
"it" · "sat"   = 2.1   (low match)
"it" · "tired" = 6.4   (medium match)
```

---

### Step 2: Scale by the Square Root of d_k

If `d_k = 64`, then `sqrt(64) = 8`. Each score is divided by 8:

```
8.5 / 8 = 1.06
3.2 / 8 = 0.40
2.1 / 8 = 0.26
6.4 / 8 = 0.80
```

**Why scale?** With large key dimensions, raw dot products become very large numbers. Large inputs push the softmax function into regions where gradients become extremely small, which causes learning to stall. Dividing by `sqrt(d_k)` keeps the values in a numerically stable range.

---

### Step 3: Apply Softmax

Softmax converts the scaled scores into a probability distribution that sums to 1:

```
"cat"   -> 0.65   (highest attention)
"tired" -> 0.20
"mat"   -> 0.10
"sat"   -> 0.05
```

---

### Step 4: Multiply by the Values

The final output is a weighted blend of all value vectors:

```
Output = 0.65 × value("cat")
       + 0.20 × value("tired")
       + 0.10 × value("mat")
       + 0.05 × value("sat")
```

The word "it" now carries mostly the information from "cat" — which is exactly the correct coreference resolution.

# Scaled Dot-Product Attention and Multi-Head Attention

## Example

We will use the following sentence throughout:

> "The cat sat on the mat because it was tired"

---

# 1. Scaled Dot-Product Attention

This section explains the left part of the diagram, read from bottom to top.

---

## Step 1 — Matrix Multiplication (Q × K)

- Query (Q): representation of the current word (e.g., "it")
- Keys (K): representations of all words

Each query is compared with every key using matrix multiplication:

| Pair              | Score |
|-------------------|-------|
| "it" vs "cat"     | 8.5   |
| "it" vs "mat"     | 3.2   |
| "it" vs "sat"     | 2.1   |
| "it" vs "tired"   | 6.4   |

This creates a compatibility score matrix.

---

## Step 2 — Scaling

The scores are scaled by:

 $$
\frac{1}{\sqrt{d_k}}
$$

If $d_k = 64$, then:

$$
\sqrt{64} = 8
$$

Scaled values:

| Original | Scaled |
|----------|--------|
| 8.5      | 1.06   |
| 3.2      | 0.40   |
| 2.1      | 0.26   |
| 6.4      | 0.80   |

### Why scaling is needed:

- Prevents extremely large values
- Keeps softmax in a stable range
- Avoids vanishing gradients

---

## Step 3 — Masking (Optional)

Masking is used only in the decoder.

- Future positions are set to \( -\infty \)
- After softmax, these become 0

This ensures:

> A position cannot attend to future tokens

The encoder does not use masking.

---

## Step 4 — Softmax

Scores are converted into probabilities:

| Word   | Weight |
|--------|--------|
| cat    | 0.65   |
| tired  | 0.20   |
| mat    | 0.10   |
| sat    | 0.05   |
| Total  | 1.00   |

These weights determine importance.

---

## Step 5 — Weighted Sum with Values (V)

The final output is computed as:


$$
\text{Output} = \sum (\text{weight} \times V)
$$



Example:

$$
0.65 \cdot V(\text{cat}) + 0.20 \cdot V(\text{tired}) + 0.10 \cdot V(\text{mat}) + 0.05 \cdot V(\text{sat})
$$

The resulting representation of "it" is dominated by "cat".
