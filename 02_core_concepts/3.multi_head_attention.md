# Multi-Head Attention

### Why Multiple Heads?

A single attention pass captures one type of relationship. Multi-head attention runs several attention operations in parallel, allowing the model to simultaneously attend to information from different representational subspaces — for instance, grammatical structure, semantic similarity, and positional relationships all at once.

### Dimensions

The original Transformer paper uses the following configuration:

- `d_model = 512` (total model dimension)
- Number of heads `h = 8`
- Per-head dimension: `d_k = d_v = 512 / 8 = 64`

Each head receives a 64-dimensional slice of the full 512-dimensional representation and performs its own independent attention operation.

### What Each Head Learns

| Head | d_k | d_v | Example of What It Captures |
|---|---|---|---|
| Head 1 | 64 | 64 | Subject-verb grammatical links |
| Head 2 | 64 | 64 | Who does what to whom |
| Head 3 | 64 | 64 | Positional relationships |
| Head 4 | 64 | 64 | Semantic similarity |
| ... | 64 | 64 | ... |
| Head 8 | 64 | 64 | Long-range dependencies |

### Concatenation and Projection

After all heads compute their outputs, the results are concatenated and projected back to the full model dimension:

```
[Head_1(64) | Head_2(64) | ... | Head_8(64)] = 512 dimensions
```

The total computational cost is equivalent to a single attention operation over the full 512-dimensional space, but the model gains eight independent perspectives on the input.

---

## Summary

The attention mechanism follows a clean four-part logic:

1. **Q (Query)** asks the question: "which words are relevant to me?"
2. **K (Key)** provides the match signal: "this is what I represent."
3. The **compatibility function** (dot product) scores how well each Q-K pair aligns.
4. **V (Value)** delivers the actual information, weighted by those scores to form the output.

Scaling stabilizes training. Multiple heads allow the model to capture different types of relationships simultaneously. Together, these components form the core of the Transformer architecture.

# Multi-Head Attention: Detailed Explanation

## 1. Core Idea

Instead of performing attention once, the model performs it **multiple times in parallel**.

- Number of heads = \( h \)
- Each head learns a **different type of relationship**
- Outputs are combined to form a richer representation

---

## 2. Why Multiple Heads?

A single attention head produces:

- One weighted average of all relationships  
- A blended representation (less precise)

Multi-head attention allows:

- Separate learning of different relationships  
- Sharper, more specialized representations  

---

## 3. Example Setup

Consider the sentence:

> "The animal didn't cross the street because it was too tired"

Model configuration:

- $d_{\text{model}} = 512$
- Number of heads $h = 8$
- $d_k = d_v = 64$


---

## 4. Step-by-Step Process

### Step 1 — Linear Projections

The input Query, Key, and Value vectors (512 dimensions) are projected into lower dimensions for each head.

For Queries:

$$Q \rightarrow Q_i = Q \cdot W^Q_i$$


Each projection:

- $W^Q_i \in \mathbb{R}^{512 \times 64}$
- Output $Q_i \in \mathbb{R}^{64}$

Similarly:

- $K \rightarrow K_i = K \cdot W^K_i$
- $V \rightarrow V_i = V \cdot W^V_i$

Each head learns its own projection matrices.

---

### Step 2 — Parallel Attention Computation

Each head independently performs scaled dot-product attention:

$$\text{head}_i = \text{Attention}(Q_i, K_i, V_i)$$

Each output is a **64-dimensional vector**.

---

### Step 3 — What Each Head Learns

Each head focuses on different relationships:

| Head | Type of Relationship |
|------|---------------------|
| 1    | Grammatical subject |
| 2    | Cause-effect        |
| 3    | Verb relationships  |
| 4    | Positional proximity|
| 5    | Semantic similarity |
| 6    | Negation context    |
| 7    | Long-range dependencies |
| 8    | Syntactic structure |

Example interpretations for the word **"it"**:

- Subject reference → "animal"  
- Cause → "tired"  
- Action → "cross"  
- Nearby noun → "street"  

---

### Step 4 — Concatenation

All head outputs are combined:

$$\text{Concat}(\text{head}_1, \text{head}_2, \dots, \text{head}_8)$$

Result:

$$64 \times 8 = 512 \text{ dimensions}$$


---

### Step 5 — Final Linear Projection

The concatenated output is passed through a final linear layer:

$$\text{Output} = \text{Concat}(\dots) \cdot W_O$$

Where:

- $W_O \in \mathbb{R}^{512 \times 512}$


This step combines all head outputs into a unified representation.

---

## 5. Full Formula

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \dots, \text{head}_h) \cdot W_O$$

Where:

$$\text{head}_i = \text{Attention}(Q W^Q_i, K W^K_i, V W^V_i)$$


---

## 6. Why Not Use a Single Head?

### Single Head (512 dimensions)

Produces one combined attention distribution:

| Word   | Weight |
|--------|--------|
| animal | 0.35   |
| tired  | 0.25   |
| street | 0.20   |
| cross  | 0.12   |
| didn't | 0.08   |

- All relationships are mixed together  
- Results in a **blended representation**

---

### Multi-Head Attention (8 heads of 64 dimensions)

Each head specializes:

| Head Type | Focus | Example |
|----------|------|--------|
| Grammar  | Subject | animal → high weight |
| Cause    | Reason  | tired → high weight |
| Position | Nearby  | street → high weight |

- Each head produces a **focused signal**
- No mixing of unrelated information

---

## 7. Comparison


| Feature                  | Single Head           | Multi-Head Attention      |
|--------------------------|------------------------|---------------------------|
| Number of attentions     | 1                      | h (e.g., 8)               |
| Dimension per attention  | 512                    | 64                        |
| Relationships captured   | Blended                | Specialized               |
| Signal clarity           | Low                    | High                      |
| Computational cost       | Same                   | Same (parallelized)       |
| Output dimension         | 512                    | 512                       |

# Multi-Head Attention: Formula and Worked Example

## The Formula

Multi-head attention is defined by two lines:

**Line 1 — The main formula:**

$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h) \cdot W^O$$

In plain terms: run `h` attention heads in parallel, concatenate all their outputs, then project the result through a final learned matrix `W_O`.

**Line 2 — What each head does:**

$$\text{head}_i = \text{Attention}(Q \cdot W^Q_i, \ K \cdot W^K_i, \ V \cdot W^V_i)$$

Each head receives the same Q, K, and V as input, but multiplies them by its own learned projection matrices before computing attention. This gives each head a distinct, independently learned perspective on the same input.

---

## The Projection Matrices

The notation `W ∈ R^(a×b)` simply means a matrix of real numbers with `a` rows and `b` columns. Using the standard paper dimensions (`d_model = 512`, `h = 8`, `d_k = d_v = 64`):

| Matrix | Dimensions | Shape |
|---|---|---|
| W_Q_i | R^(d_model × d_k) | 512 × 64 |
| W_K_i | R^(d_model × d_k) | 512 × 64 |
| W_V_i | R^(d_model × d_v) | 512 × 64 |
| W_O | R^(h·d_v × d_model) | 512 × 512 |

The `R` simply denotes real numbers — there is nothing more to it than a grid of numbers with the specified dimensions.

---

## Full Worked Example

**Task:** Encode the sentence "The cat sat" for translation into Hindi.

**Configuration:**

```
d_model = 512
h       = 8 heads
d_k     = d_v = 512 / 8 = 64
```

---

### Step 1: Input Q, K, V

Each of the three words is represented as a 512-dimensional vector. The full input matrices have shape `(3 × 512)`:

```
Q = ["The"(512), "cat"(512), "sat"(512)]   shape: 3 × 512
K = same
V = same
```

---

### Step 2: Project for Each Head

Each head applies its own learned projection matrices to produce lower-dimensional Q, K, V slices:

**Head 1:**
```
Q × W_Q_1  =  (3×512) × (512×64)  =  Q1  shape: (3×64)
K × W_K_1  =  (3×512) × (512×64)  =  K1  shape: (3×64)
V × W_V_1  =  (3×512) × (512×64)  =  V1  shape: (3×64)
```

**Head 2:**
```
Q × W_Q_2  =  (3×512) × (512×64)  =  Q2  shape: (3×64)
K × W_K_2  =  (3×512) × (512×64)  =  K2  shape: (3×64)
V × W_V_2  =  (3×512) × (512×64)  =  V2  shape: (3×64)
```

This repeats for all 8 heads, each with its own independently learned W matrices. The result is that each head sees the same sentence through a different 64-dimensional lens:

```
Head 1 W matrices  →  learned to focus on grammatical structure
Head 2 W matrices  →  learned to focus on semantic meaning
Head 3 W matrices  →  learned to focus on positional relationships
...
```

---

### Step 3: Run Attention on Each Head

Standard scaled dot-product attention is applied within each head independently:

```
head_1 = Attention(Q1, K1, V1)  →  output: (3×64)
head_2 = Attention(Q2, K2, V2)  →  output: (3×64)
head_3 = Attention(Q3, K3, V3)  →  output: (3×64)
...
head_8 = Attention(Q8, K8, V8)  →  output: (3×64)
```

For the word "cat", each head finds different relationships:

| Head | Focus | What "cat" attends to |
|---|---|---|
| Head 1 (grammar) | Subject-verb links | "sat" strongly |
| Head 2 (meaning) | Article-noun links | "The" strongly |
| Head 3 (position) | Positional context | Position 2 in sequence |
| Head 8 (long range) | Global context | All words roughly equally |

---

### Step 4: Concatenate

The eight outputs, each of shape `(3×64)`, are concatenated along the last dimension:

```
Concat(head_1, head_2, ..., head_8)
  = [(3×64) | (3×64) | ... | (3×64)]
  = (3 × 512)
```

For the word "cat" specifically, this looks like:

```
[ head1_64_dims | head2_64_dims | ... | head8_64_dims ]
  grammar info    meaning info         long-range info
  ──────────────────────────────────────────────────
                 512 dimensions total
```

The model is back to `d_model = 512` — the same dimensionality it started with.

---

### Step 5: Final Projection Through W_O

```
(3×512) × W_O(512×512)  =  Final Output (3×512)
```

`W_O` is not a random blend. It is a learned matrix that intelligently mixes the contributions of all 8 heads — synthesising what matters most from each perspective into a single unified representation.

---

## The Computational Cost Insight

The paper notes that because each head operates on a reduced dimension, the total computational cost of multi-head attention is comparable to that of a single attention head operating at full dimensionality:

```
Single-head attention:
  One attention pass with d_k = 512
  Cost ~ O(512)

Multi-head attention (h = 8):
  8 attention passes each with d_k = 64
  Cost ~ 8 × O(64) = O(512)
```

The computational budget is the same, but the model gains 8 independent perspectives on the input. That is the core efficiency argument for multi-head attention.

---

## End-to-End Data Flow

```
         Q (3×512)    K (3×512)    V (3×512)
              |            |            |
    ┌─────────┼────────────┼────────────┼──────────┐
    │ Head 1  ↓            ↓            ↓          │
    │       ×W_Q1        ×W_K1        ×W_V1        │
    │      Q1(3×64)    K1(3×64)    V1(3×64)        │
    │                 Attention                     │
    │              head_1_out (3×64)                │
    ├───────────────────────────────────────────────┤
    │ Head 2  →  same process  →  head_2_out (3×64) │
    ├───────────────────────────────────────────────┤
    │  ...                                          │
    ├───────────────────────────────────────────────┤
    │ Head 8  →  same process  →  head_8_out (3×64) │
    └───────────────────────────────────────────────┘
                       |
                    Concat
                       |
                  (3 × 512)
                       |
                    × W_O
                       |
            Final Output (3×512)
```

---

## Summary

The multi-head attention mechanism follows five steps: the same Q, K, V are projected into 8 different 64-dimensional subspaces using learned weight matrices, independent attention is computed in each subspace, the 8 outputs are concatenated back to 512 dimensions, and a final projection matrix `W_O` synthesises the result. The total cost equals that of a single full-dimensional attention pass, while the model captures 8 distinct types of relationships simultaneously.
