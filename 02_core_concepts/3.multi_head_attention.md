# Multi-Head Attention

### Why Multiple Heads?

A single attention pass captures one type of relationship. Multi-head attention runs several attention operations in parallel, allowing the model to simultaneously attend to information from different representational subspaces — for instance, grammatical structure, semantic similarity, and positional relationships all at once.

### Dimensions

The original Transformer paper uses the following configuration:

- `d_model = 512` (total model dimension)
- Number of heads `h = 8`
- Per-head dimension: `d_k = d_v = 512 / 8 = 64`

Each head receives a 64-dimensional slice of the full 512-dimensional representation and performs its own independent attention operation.

### What Each Head Learns

| Head | d_k | d_v | Example of What It Captures |
|---|---|---|---|
| Head 1 | 64 | 64 | Subject-verb grammatical links |
| Head 2 | 64 | 64 | Who does what to whom |
| Head 3 | 64 | 64 | Positional relationships |
| Head 4 | 64 | 64 | Semantic similarity |
| ... | 64 | 64 | ... |
| Head 8 | 64 | 64 | Long-range dependencies |

### Concatenation and Projection

After all heads compute their outputs, the results are concatenated and projected back to the full model dimension:

```
[Head_1(64) | Head_2(64) | ... | Head_8(64)] = 512 dimensions
```

The total computational cost is equivalent to a single attention operation over the full 512-dimensional space, but the model gains eight independent perspectives on the input.

---

## Summary

The attention mechanism follows a clean four-part logic:

1. **Q (Query)** asks the question: "which words are relevant to me?"
2. **K (Key)** provides the match signal: "this is what I represent."
3. The **compatibility function** (dot product) scores how well each Q-K pair aligns.
4. **V (Value)** delivers the actual information, weighted by those scores to form the output.

Scaling stabilizes training. Multiple heads allow the model to capture different types of relationships simultaneously. Together, these components form the core of the Transformer architecture.

# 2. Multi-Head Attention

This section explains the right part of the diagram.

---

## Step 1 — Inputs

- Query (Q), Key (K), Value (V) enter the module
- Same inputs are used across multiple heads

---

## Step 2 — Linear Projections

Each of Q, K, V is projected into lower-dimensional spaces:

$$
\begin{aligned}
Q (512) &\rightarrow Q' (64) \\
K (512) &\rightarrow K' (64) \\
V (512) &\rightarrow V' (64)
\end{aligned}
$$


This is done separately for each head.

---



### Purpose:

Each head learns a different representation or perspective.

---

## Step 3 — Parallel Attention Computation

Scaled dot-product attention is applied independently in each head.

For 8 heads:

- Head 1 → output₁ (64 dimensions)
- Head 2 → output₂ (64 dimensions)
- ...
- Head 8 → output₈ (64 dimensions)

Each head captures different relationships.

---

## Step 4 — Concatenation

All outputs are combined:

$$
[\text{output}_1 \mid \text{output}_2 \mid \dots \mid \text{output}_8]
$$

Result:

$$
64 \times 8 = 512 \text{ dimensions}
$$


---

## Step 5 — Final Linear Layer

The concatenated output is passed through a final linear layer:

$$
512 \rightarrow 512
$$


This integrates all heads into a unified representation.

---

# 3. Comparison

| Feature                  | Scaled Dot-Product Attention | Multi-Head Attention        |
|--------------------------|------------------------------|-----------------------------|
| Number of attentions     | 1                            | h (e.g., 8)                 |
| Input dimension          | Full \( d_k \)               | Split across heads          |
| Output                   | Single representation        | Multiple merged representations |
| Masking                  | Optional                     | Optional (decoder only)     |
| Computational cost       | Base                         | Similar (parallelized)      |

---

# 4. Key Insight

Multi-head attention provides multiple perspectives on the same data.

- Each head specializes in a different type of relationship
- Outputs are combined into a richer representation
- Computational cost remains similar due to parallelization

---

# 5. One-Line Summary

Scaled dot-product attention computes relevance between words, and multi-head attention extends this by learning multiple representations in parallel and combining them into a single, richer output.
