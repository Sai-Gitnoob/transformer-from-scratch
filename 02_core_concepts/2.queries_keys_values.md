# Understanding Attention in Transformers

## What Is Attention?

Attention is a mechanism that maps a query and a set of key-value pairs to an output, where all of these are vectors. The output is computed as a **weighted sum** of the values, where the weight assigned to each value is determined by a compatibility function applied to the query and its corresponding key.

To make this concrete: when processing the word "it" in a sentence, the model needs to figure out what "it" refers to. Does "it" attend more to "the cat", "the road", or some other word? The attention weights answer exactly this question.

---

## The Compatibility Function

The compatibility function answers a single question: **how well does this query match this key?**

Think of it like a search engine ranking scores. Given the query "I want space books", the compatibility function scores each key:

```
Query:  "I want space books"
Key 1:  "astronomy and planets"  → score = 0.90  (highly relevant)
Key 2:  "cooking recipes"        → score = 0.02  (not relevant)
Key 3:  "space history"          → score = 0.85  (highly relevant)
```

These scores become the attention weights. A high score means the model pays more attention to the corresponding value. In Transformers, the compatibility function is simply the **dot product** between Q and K: the more similar two vectors are in direction, the higher their dot product.

---

## The Library Analogy: Q, K, and V

It helps to think of attention in terms of a library search:

| Concept | Library Analogy | In Attention |
|---|---|---|
| Query (Q) | Your search request: "I want books about space" | The current word asking "who is relevant to me?" |
| Key (K) | Each book's index tag: "this book is about space" | Each word advertising "this is what I contain" |
| Value (V) | The actual content inside the book | The actual information the word carries |
| Output | The books you actually read | A weighted blend of all values |
